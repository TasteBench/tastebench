# Kaggle Unsupervised Competition

Generate datasets for the Kaggle competition where participants predict which alternative protein product was rated as more similar to its animal-based counterpart.

## How to regenerate the dataset

Run both commands from the project root:

```bash
# Step 1: Generate CSVs (run all cells in the notebook)
jupyter nbconvert --to notebook --execute kaggle_tastebench/generate_data/generate_kaggle_datasets.ipynb

# Step 2: Add images and update products.csv with image_path column
python kaggle_tastebench/generate_data/add_images_to_dataset.py
```

**Step 1** produces all CSV files in `dataset/`.

**Step 2** does the following:
1. Copies NECTAR product images (renamed to anonymized codes) into `dataset/images/`
2. Generates synthetic images for Taste Like products
3. Drops some Taste Like images to match NECTAR's missing-image rate per category
4. Adds an `image_path` column to `dataset/products.csv`

Options: `--dry-run` (preview without writing), `--seed N` (default: 42)

The script is fully deterministic — running it twice produces identical outputs.

## Image construction

### NECTAR products
Original photographs are copied without modification under anonymized product codes. One view per category is selected (e.g., `cross_section_front` for Nuggets, `cooked_front` for Burgers). NECTAR products lacking source photographs receive no image.

### Taste Like products
No real images exist for Taste Like products. To prevent participants from identifying product sources via image availability, synthetic images are generated by augmenting NECTAR source images from the same category.

The augmentation pipeline applies 10 sequential transformations designed to mimic natural camera/photography variation:

| Transformation | Parameters | Purpose |
|---|---|---|
| Horizontal flip | 50% chance | Reverses orientation |
| Random crop + resize | 8-18% removed | Changes spatial framing |
| Color temperature | ±15 on R/B channels | Warm/cool white balance |
| White balance | ±10 on G channel | Green-magenta axis |
| Gamma correction | 0.85-1.15 | Camera exposure curves |
| Brightness | 0.82-1.18 | Lighting variation |
| Contrast | 0.82-1.18 | Tonal range variation |
| Saturation | 0.80-1.20 | Color intensity variation |
| Gaussian noise | sigma 2-5 | Sensor noise simulation |
| Gaussian blur | radius 0.2-0.5 | Masks compression differences |

Output JPEG quality is varied randomly between 75-92 per image.

Each augmentation individually falls within natural photographic variation. Their composition resists casual visual matching without producing artifacts characteristic of heavy augmentation. The horizontal flip is 50/50 rather than always-on to prevent flip direction from serving as a classification signal.

### Missing-image balancing
Some NECTAR products lack source photographs. To prevent missing images from serving as a source-identification signal, Taste Like images are randomly dropped at a proportional rate within each affected category, producing comparable missing rates per category across both sources.

## Uploading to Kaggle

Upload everything in `dataset/` **except** `product_code_map.csv` (it reveals which products are NECTAR vs Taste Like).

| File | Upload as |
|---|---|
| `products.csv` | Competition data |
| `ranking_pairs.csv` | Competition data |
| `sample_submission.csv` | Competition data |
| `images/` | Competition data |
| `solution.csv` | Solution (uploaded separately on Kaggle) |
| `product_code_map.csv` | **Do not upload** — internal only |

## Supporting files

- `taste_like_category_map.py` — maps Taste Like product slugs to NECTAR categories (imported by the notebook)
