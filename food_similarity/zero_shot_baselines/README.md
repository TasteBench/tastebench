# Unsupervised Pairwise Ranking

Predicts which of two products is more similar to its animal-based counterpart,
using an unsupervised distance-to-centroid approach. No labeled pairs are used
for training.

## Approach

For each product category (burgers, nuggets, etc.):

1. **Extract features** per product from up to 4 sources:
   - **N** (Nutrition) — category-specific nutrition columns, normalized per 100g
   - **C** (Compound) — chemical compound embeddings via FoodAtlas + FART model (768-dim)
   - **T** (Text) — ingredient text embeddings via Qwen3-Embedding (sensory-augmented)
   - **I** (Image) — product image embeddings via DINOv3 ViT-L/16 (1024-dim)
2. **Compute animal centroid** — mean feature vector of animal products in each category
3. **Preprocess** — StandardScaler per feature type, producing whitened cosine similarity
   (Mahalanobis cosine) that equalizes each dimension's contribution
4. **Score each product** — weighted combination of per-feature distances to centroid
5. **Predict pairs** — product with higher similarity score (closer to animal centroid) wins

Ingredient-level features (C, T) are aggregated to product level using
**inverse-rank weights** (1/(position+1)), where position is the ingredient's
index in the FDA-mandated descending-weight order. See
`lib/features/__init__.py` for the rationale (Zipfian decay, FDA 21 CFR 101.4).

## LLM Baseline

An alternative approach queries LLMs directly to compare product pairs.
The LLM receives raw product data (ingredients, nutrition, images) and
predicts which product is more similar to its animal-based counterpart.

- **Models**: Qwen3.5 397B A17B, Gemini 3.1 Pro Preview (via OpenRouter)
- **Ablations**: 7 feature combinations per model (ingredients, nutrition, image, and combos)
- **Anti-bias**: Deterministic product swapping (MD5 hash) to prevent position effects
- **Reasoning**: Chain-of-thought captured in response logs for qualitative analysis

Requires `OPENROUTER_API_KEY` in a `.env` file or exported in your shell.

## Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Run a single distance-based experiment (nutrition-only, fastest)
python run.py --config configs/cosine_dist/N.yaml

# Run a single LLM experiment (requires OPENROUTER_API_KEY)
python run.py --config configs/llm/qwen3_5_397b_a17b/ingredients_nutrition.yaml

# Run all distance-based configs (30 total)
python run.py --all

# Compare two result CSVs (agreement rate, Cohen's kappa, per-category)
python compare_results.py results/cosine_dist/cosine_N.csv results/l2_dist/l2_N.csv
```

Distance results are saved to `results/{experiment_name}.csv` with a copy of the config.
LLM results are saved to `results/llm/{model}/submissions/`, `logs/`, and `configs/`.

## Prerequisites

### Data

The following must exist before running:

| Path | Description | Required for |
|------|-------------|--------------|
| `data/competition/products.csv` | Product metadata and nutrition | All configs |
| `data/competition/ranking_pairs.csv` | Test pairs to predict | All configs |
| `data/product_labels_manually_cleaned.csv` | Labeled products with cleaned ingredients | All configs |
| `data/competition/images/` | Product images (~8.6 GB) | I (image) configs |

### Pre-computed Caches

These caches live in `../../shared/data/caches/` and must be generated before
running configs that use compound or text features. **Inference is cache-only** —
`run.py` never makes network calls, so all caches must be built beforehand.

Run the cache preparation scripts **in this order** from the repo root:

```bash
# Step 1: Resolve PubChem CIDs → SMILES (calls PubChem API).
#         Resume-safe: re-run to retry any 503/timeout failures.
python shared/scripts/prepare_caches/prepare_smiles_cache.py

# Step 2: Resolve ChEBI IDs → SMILES (calls PubChem API).
python shared/scripts/prepare_caches/resolve_chebi_smiles.py

# Step 3: Generate FART embeddings from cached SMILES.
#         Requires steps 1-2 to be complete.
python shared/scripts/prepare_caches/prepare_fart_embeddings.py

# Step 4 (optional): Generate LLM sensory descriptions for ingredients.
#         Requires OPENROUTER_API_KEY in .env.
python shared/scripts/generate_sensory_descriptions.py
```

| Cache | Size | Generated by | Required for |
|-------|------|--------------|--------------|
| `smiles_cache.csv` | ~13 MB | Step 1 | C configs |
| `chebi_smiles_cache.csv` | ~50 KB | Step 2 | C configs |
| `fart_compound_embeddings.pkl` | ~430 MB | Step 3 | C configs |
| `sensory_descriptions.csv` | ~126 KB | Step 4 | T configs (optional) |

## Config System

### Distance-based configs

Configs live in `configs/{distance_metric}/` with names encoding enabled features:

- **N** = Nutrition, **C** = Compound embeddings, **T** = Text embeddings, **I** = Image embeddings
- Example: `NCTI.yaml` enables all four; `N.yaml` enables nutrition only

Two distance metrics: `cosine_dist/` (15 configs) and `l2_dist/` (15 configs).

Each config specifies per-feature parameters (scaler, PCA, model, weights) and
global settings (distance metric, normalization method, random seed). See any
YAML file for the full schema.

### LLM configs

Configs live in `configs/llm/{model_name}/` with names encoding prompt features:

- `ingredients.yaml`, `nutrition.yaml`, `image.yaml` — single-feature ablations
- `ingredients_nutrition.yaml`, `ingredients_image.yaml`, `nutrition_image.yaml` — pairs
- `ingredients_nutrition_image.yaml` — all features

Two models: `qwen3_5_397b_a17b/` and `gemini_3_1_pro_preview/` (7 configs each).

## Output

### Distance-based experiments
- `results/{name}.csv` — submission file (`test_id, higher_rated_product`)
- `results/{name}.yaml` — copy of the config used

### LLM experiments
- `results/llm/{model}/submissions/{features}.csv` — submission file
- `results/llm/{model}/logs/{features}.csv` — response log (raw response, reasoning, latency, token usage)
- `results/llm/{model}/configs/{features}.yaml` — copy of the config used

Diagnostic statistics (score distribution, per-category breakdown) are printed
to stdout.

## File Structure

```
predict/
  run.py                    <- Entry point: load config, run experiment, save results
  compare_results.py        <- Compare two result CSVs (agreement, Cohen's kappa)
  requirements.txt          <- Python dependencies
  lib/
    data.py                 <- Data loading (products, labels, pairs)
    preprocessor.py         <- Feature preprocessing (scaling, PCA)
    features/
      __init__.py           <- Feature registry + inverse-rank weighting rationale
      base.py               <- BaseFeature interface + shared ingredient weight parsing
      nutrition.py           <- Nutrition feature extraction (per-100g, category-specific)
      compound.py            <- Compound embedding aggregation (FoodAtlas + FART)
      ingredient_text.py     <- Text embedding aggregation (sensory-augmented)
      image.py              <- Image embedding extraction (DINOv3)
    models/
      __init__.py            <- Model registry
      base.py               <- BaseModel interface
      distance_predictor.py  <- Distance-to-centroid ranking model
      llm_predictor.py       <- LLM pairwise ranking model (via OpenRouter)
  configs/
    cosine_dist/             <- 15 configs using cosine distance
    l2_dist/                 <- 15 configs using L2 distance
    llm/
      qwen3_5_397b_a17b/    <- 7 Qwen3.5 ablation configs
      gemini_3_1_pro_preview/ <- 7 Gemini ablation configs
```
