# Croissant metadata

This submission ships a Croissant JSON-LD metadata file for the gated
NECTAR Kaggle Dataset. Pre-existing public datasets cited in the paper
(FoodAtlas, FooDB, FartDB, Taste Like) are owned and documented by their
upstream maintainers.

## Contents

| File | Describes | Visibility | Source |
|---|---|---|---|
| `nectar.json` | Raw NECTAR sensory ratings + ingredient/nutrition CSVs + product images. NDA-protected; reviewers access the Kaggle Dataset via the private access link in the paper. | Private | Kaggle Dataset (auto-generated, optionally augmented with RAI fields) |

The file is auto-generated by Kaggle when the corresponding dataset is
published, then committed here so the supplementary submission zip
satisfies the FAQ requirement that *"a Croissant file for every dataset
... combined in a .zip folder"* be included.

## Existing datasets that don't need a Croissant here

The following are pre-existing public datasets cited in the paper. The
track FAQ is explicit: *"involves using existing public datasets ... No
[hosting needed]."* The originating projects own their Croissant
metadata.

- **FoodAtlas v4.0** — fetched by `data/download_public.sh`
- **FartDB** — vendored splits at `molecular/data/splits/`
- **FooDB 2020-04-07** — fetched by `data/download_public.sh`

(Taste Like is now distributed inside Bundle B alongside the gated
NECTAR data; it appears as a co-source in `nectar.json`.)

## Validating a Croissant file

```bash
pip install mlcroissant
mlcroissant validate --jsonld croissant/nectar.json
```

The file must validate with no errors before the final submission.

## Augmenting Kaggle's auto-generated Croissant with RAI fields

Kaggle's auto-Croissant covers the core fields (FileObject, RecordSet,
column types). The track also requests Responsible AI (RAI) fields:
collection methodology, demographic representation, known biases,
intended use, unintended use. Add these by hand under a top-level
`"rai:"` block in each JSON file. See the
[Croissant RAI extension spec](https://docs.mlcommons.org/croissant/docs/croissant-rai-spec.html)
for the exact field names and value ranges.

For NECTAR specifically, the RAI fields worth including:
- **Collection methodology**: NECTAR's BIBD design across two
  separate-day sessions, untrained omnivore consumer panelists.
- **Demographic representation**: panelist demographics not collected
  beyond omnivore self-identification; geographic distribution biased
  toward NECTAR's testing locations.
- **Known biases**: brand-privacy obfuscation may bias which products
  appear in cover; Taste Like cover products lack sensory ratings.
- **Intended use**: training and evaluation of food-similarity ranking
  models for sustainable-protein product development.
- **Unintended use**: should not be used to publicly identify which
  brands underperformed in NECTAR sensory testing.
